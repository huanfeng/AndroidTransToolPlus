### 2.2 翻译系统架构

#### 翻译流程

```
┌─────────────────┐
│  用户触发翻译   │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  TranslationStore │
│   batchTranslate  │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  OpenAI翻译器   │
│ batchTranslateChunked │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│   批量请求      │
│ 每批 N 条记录   │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  AI 翻译响应    │
│ JSON 格式       │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  应用翻译结果   │
│ 更新进度显示    │
└─────────────────┘
```

#### 核心组件

**1. TranslationStore** (`stores/translation.ts`)
- 管理翻译状态和进度
- 提供批量翻译接口
- 进度跟踪和统计

**2. OpenAITranslator** (`services/translation/openai.ts`)
- OpenAI API 集成
- 批量翻译实现
- 提示词管理

**3. AiModel** (`models/ai.ts`)
- AI 模型预设
- 提示词模板
- 批量翻译模板

### 2.3 批量翻译优化

#### 问题与解决方案

**问题1**: 逐条翻译速度慢、费用高
- **原因**: 每条请求都需要完整提示词，有效内容比例低
- **解决**: 批量处理，多条记录共享提示词

**问题2**: 进度显示不一致
- **原因**: 总量计算方式不统一
- **解决**: 分离内部总量和显示总量

**问题3**: 未翻译过滤逻辑不统一
- **原因**: 多个地方使用不同的过滤标准
- **解决**: 统一使用文件映射数据判断

#### 核心改进

1. **预计算总量**
   - 预先计算每个语言需要翻译的项目数
   - 分离内部总量（条目数 × 语言数）和显示总量（实际需要翻译的数量）

2. **实时进度更新**
   - 分批处理，每批完成后立即更新进度
   - 独立进度跟踪变量，确保响应式更新

3. **统一过滤逻辑**
   - 使用统一的未翻译过滤标准
   - 传递文件映射数据确保一致性

### 2.4 数据流

#### 翻译数据流

```typescript
// 1. TranslationStore.batchTranslate()
items: Map<string, ResItem>
languages: Language[]
autoUpdateTranslated: boolean
fileMap?: Map<Language, any>

// 2. 预计算阶段
languageTaskCounts: Record<string, number>  // 每个语言的总量
languageActualCounts: Record<string, number>  // 每个语言实际需要翻译的数量

// 3. 翻译阶段
batchItems: Array<{
  key: string
  text: string | string[]
  context?: string
}>

// 4. 应用结果
tasks: TranslationTask[]
progress: {
  total: number      // 显示用总量
  completed: number  // 已完成数量
  failed: number     // 失败数量
  percentage: number // 百分比
}
```

## 3. 关键技术特性

### 3.1 平台适配层

平台适配层提供统一的浏览器 API 接口，基于 File System Access API 实现文件系统操作。

**核心接口**:
- 文件系统适配（基于 File System Access API）
- 存储适配（基于 localStorage）
- 平台能力检测

### 3.2 翻译引擎

**支持的模型**:
- gpt-5-mini (默认推荐)
- gpt-5 (标准模型)
- gpt-5-nano (价格便宜)
- gpt-4.1-mini
- gpt-4o-mini
- 自定义模型

**提示词优化**:
- 单条翻译模板
- 批量翻译模板（包含上下文）
- 模板渲染机制

### 3.3 状态管理

**Pinia Stores**:

1. **ConfigStore** - 应用配置
2. **ProjectStore** - 项目数据
3. **TranslationStore** - 翻译状态
4. **LogStore** - 日志管理
